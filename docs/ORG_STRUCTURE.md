# ORG_STRUCTURE

## 1. 문서 목적
VOE Listening Platform에서 사용하는 조직 구조, 조직별 보드 운영 방식, 상하위 조직 집계 로직, 향후 권한 확장 방향을 정의한다.

이 플랫폼은 단일 전사 게시판이 아니라, **조직 맥락 속에서 의견을 청취하고 해석하는 구조**를 전제로 한다.

---

## 2. 조직 구조 원칙

### 2.1 기본 원칙
- 각 조직은 고유한 게시판 단위가 된다.
- 의견은 반드시 하나의 조직에 귀속된다.
- 상위 조직은 하위 조직의 의견을 집계해서 볼 수 있다.
- 하위 조직은 자신의 의견만 본다.
- 분석과 인사이트는 조직 단위로 필터링 가능해야 한다.
- 전사 관점에서는 모든 조직 데이터를 통합해 볼 수 있다.

### 2.2 설계 철학
- 구성원 입장: “내 의견이 어디로 가는지 안다.”
- 관리자 입장: “어느 조직에서 어떤 신호가 반복되는지 본다.”
- HR 입장: “패턴을 조직 실행 단위와 연결한다.”

즉, **의견의 수집 단위와 실행의 단위를 맞추는 것**이 핵심이다.

## 4. 집계 로직
- 상위 조직을 선택하면 모든 하위 조직의 의견을 포함해 집계한다.
- 하위 조직을 선택하면 해당 조직 의견만 보여준다.
- 대시보드/인사이트는 동일한 집계 규칙을 사용한다.

---

## 5. 보드 라우팅 모델
- `/boards` 조직 보드 목록
- `/boards/[orgSlug]` 개별 조직 보드
- 상위 조직 보드에서는 하위 조직 요약이 추가로 제공된다.

---

## 6. 권한 확장 방향(향후)
- 조직 기반 역할: 직원/관리자/리더
- 조직 범위 기반 접근 제어(상위 조직 권한은 하위 조직 포함)
- HR 전사 권한은 전체 데이터 접근
